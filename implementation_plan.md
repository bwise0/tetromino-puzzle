# 테트로미노 이미지 퍼즐 - 개발 계획서 (Implementation Plan)

## 📌 목표
Vite + TypeScript + PixiJS 템플릿을 사용하여 HTML5/JS 기반 웹 퍼즐 게임을 구축합니다. 플레이어는 원본 이미지가 테트로미노 모양(4개의 사각형이 연결된 형태)으로 쪼개진 퍼즐 조각들을 드래그 앤 드롭하여 원래의 이미지로 다시 조립해야 합니다.

---

## 🚀 개발 단계 제안

### 1단계: 기본 렌더링 및 프로젝트 설정
* **목표:** 개발 환경의 기본 코드를 정리하고 그래픽 렌더링을 위한 캔버스를 화면에 띄웁니다.
* **추진 항목:**
  1. 제공된 보일러플레이트 코드([main.ts](file:///c:/Users/bwise/Desktop/tetromino/tetromino-puzzle/src/main.ts), [style.css](file:///c:/Users/bwise/Desktop/tetromino/tetromino-puzzle/src/style.css)) 초기화 빛 비우기
  2. PixiJS의 `Application` 인스턴스를 생성하고 DOM(문서 객체 모델)에 캔버스 추가
  3. PixiJS `Texture`를 활용하여 원본이 될 테스트용 이미지 로드 및 화면 출력 확인

### 2단계: 보드 그리드 및 테트로미노 조각 생성
* **목표:** 게임 보드를 그리드(격자)로 나누고 개별 퍼즐 조각 객체를 생성합니다.
* **추진 항목:**
  1. **그리드 설정:** 타일 1개의 크기(예: 64x64 픽셀)와 전체 보드의 행/열 크기 정의
  2. **모양 정의:** 표준 테트로미노 모양(I, J, L, O, S, T, Z)을 2차원 배열/행렬로 정의
  3. **이미지 분할:** 정의된 모양에 따라 원본 이미지에 마스킹(Masking) 처리를 하여 각 PixiJS `Sprite` 또는 `Graphics` 객체를 생성
  4. 생성된 퍼즐 조각들을 "퍼즐 보드" 근처의 임의의 "대기열(플레이 영역)"에 무작위로 배치

### 3단계: 상호작용 및 기본 로직 (Interaction)
* **목표:** 플레이어가 마우스나 터치로 퍼즐 조각을 잡고, 이동하고, 회전할 수 있도록 만듭니다.
* **추진 항목:**
  1. 각 조각 객체에 상호작용 활성화 (`obj.eventMode = 'dynamic'`)
  2. 마우스 포인터 이벤트(`pointerdown`, `pointermove`, `pointerup`)를 활용하여 드래그 앤 드롭(Drag & Drop) 구현
  3. 우클릭 또는 더블클릭을 감지하여 퍼즐 조각을 90도 회전(Rotation)시키는 기능 구현
  4. 클릭된(선택된) 조각이 다른 조각들에 가려지지 않도록 화면의 맨 앞(z-index 최상단)으로 렌더링 순서 변경

### 4단계: 그룹화(Merging) 및 게임 승리 상태
* **목표:** 게임의 핵심 메커니즘인 '조각 병합'과 승리 판별 로직을 완성합니다.
* **추진 항목:**
  1. **스내핑(Snapping):** 조각의 드래그를 놓았을 때 주변에 올바른 그리드 칸이 있다면, 정중앙으로 자석처럼 달라붙게 하는 처리
  2. **충돌 처리:** 퍼즐 보드 위에서 다른 조각과 위치 및 셀이 겹치지 않게 방지
  3. **그룹화 (Merge):** 내려놓은 조각이 원래 이미지 상에서 인접해야 하는 또 다른 올바른 조각과 닿았을 때(비율, 위치, 회전이 일치할 때), 이 둘을 묶어 하나의 큰 조각 그룹으로 만듦 (이후 같이 움직임)
  4. **승리 조건:** 모든 조각들이 성공적으로 하나로 맞춰져 원본 이미지가 완전히 복구되면 게임 클리어 판정

### 5단계: 폴리싱(Polish) 및 UI 구현
* **목표:** 게임의 완성도를 높이고 필요한 유저 인터페이스를 제공합니다.
* **추진 항목:**
  1. HTML/CSS를 활용하여 캔버스 위에 덮어씌워지는 형태의 UI 구현 (타이머, 남은 조각 수, 이동 횟수 등)
  2. "다시 시작(Restart)", "난이도 선택(조각을 더 작게 쪼개기)" 버튼 구현
  3. 간단한 시각적 피드백 효과(예: 올바르게 조각이 결합될 때 살짝 빛나는 연출) 및 효과음 적용

## ✅ 검증 방식 (Verification Plan)
### 자동화 테스트 (Automated Tests)
* 초기 프로토타이핑 특성상 즉각적인 자동화 테스트 프레임워크 도입은 필수가 아닙니다. 다만 행렬 회전 로직이나 인접 좌표 판별 로직 등은 점진적으로 단위 테스트(Unit Test)를 작성할 수 있습니다.

### 수동 검증 (Manual Verification)
1. 터미널에서 `npm run dev` 명령어를 실행합니다.
2. 브라우저로 로컬 개발 서버에 접속합니다.
3. 캔버스가 정상적으로 렌더링 되는지 확인합니다.
4. 조각들이 드래그되고 정상적으로 회전하는지 점검합니다.
5. 올바른 위치에 접근할 때 자석처럼 붙고(Snapping), 조각들이 합쳐진 뒤로 게임 승리 상태까지 무사히 도달하는지 플레이를 통해 검증합니다.
